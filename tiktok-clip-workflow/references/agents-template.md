# AGENTS.md 模板

新建项目时把这份模板复制到项目根目录，命名为 `AGENTS.md`。Codex / Claude Code / Mavis 会自动读取，跑本工作流时按这份配置执行。

---

```markdown
# AGENTS.md

## 项目定位

这是一个抖音/短视频自动化剪辑项目，基于 tiktok-clip-workflow Skill。
主要用途：<电商带货 / 混剪 Vlog / 营销推广 / ...>

## 视频规格

- 分辨率：1080x1920（9:16 竖版）
- 时长：45-90 秒（抖音带货最佳区间）
- 码率：8-12 Mbps
- 帧率：30 fps

## 输出目录

- 素材库：`./assets/`
- 工作区：`./work/<YYYY-MM-DD>/`
- 剪映草稿输出：`~/Movies/JianyingPro/User Data/Projects/com.lveditor.draft/`

## 素材库分类约定

`assets/` 下的子目录用途：

| 目录 | 内容 | 备注 |
| --- | --- | --- |
| `assets/外观/` | 产品外观特写、包装展示 | 主推产品视觉 |
| `assets/功能/` | 产品使用、功能演示 | 讲卖点用 |
| `assets/场景/` | 使用场景、生活场景 | 情绪铺垫 |
| `assets/人物/` | 真人出镜、口播 | 谨慎使用 |
| `assets/AI生成/` | AI 生成素材 | 补齐真实素材缺口 |
| `assets/BGM/` | 背景音乐 | 可选 |

## 配音配置

- 默认音色：`BV700_streaming`（灿灿）
- 语速：`1.0`（默认）
- 采样率：`24000`

## 匹配阈值

- 高置信度：`≥ 0.75` 直接采用
- 中置信度：`0.60 - 0.75` 采用但标记降级
- 低置信度：`< 0.60` 标记「缺素材」，不硬填

## 多样性约束（防机器感）

- 同一素材连续使用上限：2 镜
- 同类型场景连续上限：5 镜
- 同分类目录连续上限：3 镜

## 验收标准

- ✅ 参考视频每个镜头都有对应素材（或明确标记缺）
- ✅ 配音自然，无爆音、无跳字
- ✅ 剪映草稿能正常打开，素材无红叉
- ✅ 时间轴对齐（配音 = 画面）
- ✅ 字幕不遮关键内容（产品/价格/关键词）
- ✅ 总时长控制在 45-90 秒

## 禁止事项

- 🚫 移动 `assets/` 里的原始素材（一律 `cp` 不 `mv`）
- 🚫 匹配置信度 < 0.60 硬填素材
- 🚫 覆盖已有剪映草稿（新草稿必须新 UUID）
- 🚫 用完全无关的素材做主视觉
- 🚫 单条视频超过 90 秒（抖音完播率会崩）

## 常用触发指令

跟 Codex/Claude Code 说：

- 「用 tiktok-clip-workflow 处理 `work/2026-07-09/reference.mp4`」
- 「拆解参考视频」→ Skill 跑 Step 1
- 「匹配素材」→ Skill 跑 Step 2  
- 「生成配音」→ Skill 跑 Step 3
- 「生成剪映草稿」→ Skill 跑 Step 4
- 「重新匹配第 3 镜」→ 局部重跑

## 我的常用参考视频

（可选：把常用的爆款参考视频链接/结构记在这里，方便下次复用）

- 吹风机爆款结构：钩子(0-3s) → 痛点(3-8s) → 卖点(8-30s) → 场景(30-45s) → 促销(45-60s)
- 化妆品爆款结构：...
```

---

## 用法说明

1. **新项目**：把上面代码块内容复制到 `<项目根>/AGENTS.md`
2. **改配置**：直接编辑各段字段（比如换默认音色、调匹配阈值）
3. **Skill 会读取**：跑本工作流时，Skill 会先读 AGENTS.md 拿到项目约束，按你的配置执行

**建议**：每个不同类型的项目（带货 / Vlog / 营销）各建一份，参数会差得比较多。
