# TikTok Clip Workflow

## Core Rule

不是"帮你剪一个视频"，而是**自动化跑完 4 步流水线**，产出一份用户可以打开、微调、直接导出的**剪映草稿**。

核心思路：**抄爆款结构 + 复用素材库**。参考视频提供**节奏和结构**，用户素材库提供**画面**，配音和字幕自动生成，最后一键出剪映草稿。

**适用**：
- ✅ 电商带货短视频
- ✅ 混剪 Vlog / 营销推广
- ✅ 需要多段素材拼接的偏混剪类视频

**不适用**：
- ❌ 精品原创视频
- ❌ 需要精细镜头语言 / 复杂转场 / 细腻情绪的内容
- ❌ 全新原创无参考视频的情况

## Interaction Rule

不要用问卷阻塞用户。用户丢参考视频 + 素材库路径过来 → 直接开跑。

- 用户提供参考视频路径 + 素材库路径 → 自动执行 4 步流水线
- 用户没建 AGENTS.md → 先用 `references/agents-template.md` 生成一份放到项目根目录
- 用户没建目录结构 → 按 `SKILL.md` 里的标准结构自动 `mkdir`
- 拿不准的步骤（配音音色、匹配阈值）→ 用默认值先跑，出结果后让用户调
- 每步跑完必须让用户 review，不要一口气跑完 4 步不给用户看中间产物

## When to Use

任一命中就触发本 Skill：

- 用户说「抄这个爆款做视频」/「按这个参考剪一条」
- 用户丢一个视频 + 一个素材库文件夹，问"能不能批量出片"
- 用户想「批量剪带货视频」/「一天出 30 条」
- 用户明确说「用 tiktok-clip-workflow」/「按你那个抖音工作流跑」
- 用户提到「剪映草稿」/「recipe.json」/「fragment_plan.json」等本 Skill 的中间产物

**不触发**：
- ❌ 用户要做全新原创视频，没有参考视频
- ❌ 用户要做电影级 / 广告级精品，需要人工精修
- ❌ 用户只要单纯的视频转码/剪切/合并（用 FFmpeg 直接干）

## Environment Prerequisites

跑本 Skill 前先跑一次环境自检（用户第一次触发时执行）：

```bash
# 必装
ffmpeg -version                    # FFmpeg，视频拆分/合成/转码底层
python3 --version                  # Python ≥ 3.8
pip3 list | grep -E "opencv|pillow|numpy"   # 图像处理库

# 可选（配音方式二选一）
# 方式 A: 豆包语音（付费但稳定，推荐电商带货）
# 方式 B: VoxCPM（免费开源，声音克隆）
```

缺什么就用 `references/environment-setup.md` 装什么。

## Project Structure

标准项目目录（本 Skill 默认按这个结构工作）：

```
项目根目录/
├── AGENTS.md                       # 项目配置（目标/输出规格/验收标准）
├── assets/                         # 素材库（可复用）
│   ├── 外观/
│   ├── 功能/
│   ├── 场景/
│   ├── AI生成/
│   └── 真人实拍/
└── work/                           # 工作区（每条视频一个日期文件夹）
    └── 2026-07-09/
        ├── reference.mp4           # 参考视频
        ├── shots/                  # 拆分出的镜头片段
        ├── keyframes/              # 每个镜头的关键帧
        ├── recipe.json             # 参考视频结构分析
        ├── fragment_plan.json      # 每个镜头的素材需求
        ├── matches.json            # 素材匹配结果
        ├── voice/                  # 生成的配音文件
        ├── subtitles.srt           # 字幕文件
        └── draft/                  # 剪映草稿输出目录
```

用户没有这个结构时，Skill 自动创建。素材库放 `assets/`（永久），每条视频独立放 `work/日期/`。

## The 4-Step Pipeline

### Step 1: 拆解参考视频

**目的**：把参考视频拆成一个个独立镜头，提取关键帧，作为后续素材匹配依据。

**输入**：`work/<日期>/reference.mp4`

**输出**：
- `shots/shot_001.mp4`, `shot_002.mp4`, ... 拆好的镜头
- `keyframes/shot_001_key.jpg`, ... 每镜关键帧
- `recipe.json` — 参考视频结构分析（镜头列表 + 时长 + 转场 + 提取的文案）

**关键规则**：
- 用 FFmpeg 的场景检测（`scene detection`）做初步拆分，阈值 `0.3` 起
- 每个镜头提取 3 帧关键帧（开头/中间/结尾）用于视觉匹配
- 用 OCR/ASR 提取文案：先跑 ASR（音频转文字），失败再跑 OCR（字幕识别）
- 拆完后**必须让用户 review**：镜头是否切对了、文案对不对
- 用户可以让 Skill 重新调整某段（"第 3 镜切得太早了，往后挪 0.5 秒"）

详见 `references/step-1-decompose.md`。

### Step 2: 视觉匹配素材

**目的**：Skill 遍历用户素材库，给每镜头找到最匹配的素材。

**输入**：`recipe.json` + `assets/` 素材库路径

**输出**：
- `fragment_plan.json` — 每个镜头的素材需求（场景/动作/景别/避免项）
- `matches.json` — 最终匹配结果（每镜用哪个素材 + 置信度 + 是否降级）
- `work/<日期>/selected/` — 匹配到的素材**复制**（不是移动）到此

**匹配逻辑**：
- 提取每镜关键帧 + 每个素材关键帧
- 计算颜色分布、亮度、构图相似度 → 匹配得分
- 得分 ≥ 阈值 → 命中；< 阈值 → 标记「缺素材」，需要人工补

**铁律**（防生成看起来很怪的视频）：
- 🚫 **同一素材不允许连续 3 镜使用**（会显得单调）
- 🚫 **同类型场景不允许连续超过 5 镜**（观众一眼看出是机器生成）
- 🚫 **宁可留空素材，也不用无关素材硬凑**（宁缺勿滥）
- ✅ 相邻镜头视觉风格要有变化（远景 → 特写 → 中景 交替）
- ✅ 素材文件**只复制不移动**（保护原始素材库）

详见 `references/step-2-match-assets.md`。

### Step 3: 生成配音

**目的**：用 TTS 生成配音，同时按配音实际时长**反向调整**视频节奏。

**输入**：`recipe.json` 里的文案（用户可先修正）

**输出**：
- `voice/voice_001.mp3`, ... 每镜配音
- `voice/timing.json` — 每条配音的实际时长 + 对应镜头
- 更新 `matches.json` — 按配音时长调整每镜播放时长

**配音方式**：
- **A. 豆包语音大模型**（推荐电商带货）：稳定、跟剪映音色同源、1 分钟约 4-8 毛
- **B. VoxCPM**（免费开源，声音克隆）：适合需要自己声音 / 特定音色

**节奏对齐规则**：
- 配音实际读了 5 秒，但参考视频对应镜头只有 3 秒 → 该镜头拉长到 5 秒
- 配音短于原镜头 → 该镜头缩短到配音时长（保留自然结尾 0.3s）
- 无对话镜头（纯 B-roll）→ 保留参考视频原时长

详见 `references/step-3-generate-voice.md`。

### Step 4: 生成剪映草稿

**目的**：把前 3 步的产物打包成一份**剪映可以直接打开**的草稿文件。

**输入**：`matches.json` + `voice/` + 字幕

**输出**：
- `draft/<草稿名>/` — 剪映草稿目录（可以直接打开）
- 草稿包含：视频轨（匹配素材）+ 音频轨（配音）+ 字幕轨 + 时间轴对齐

**剪映草稿铁律**（不遵守就打不开或覆盖已有草稿）：
- ✅ 三个关键 ID **必须一致且不重复**：`content_id` / `metadata_id` / `root_index_id`
- ✅ 素材路径**必须绝对路径**，且文件必须真实存在
- ✅ 音频轨、视频轨、字幕轨的**时间轴严格对齐**（毫秒级）
- 🚫 不允许移动素材文件（草稿引用绝对路径，移动就打不开）
- 🚫 不允许 ID 跟已有草稿冲突（会覆盖之前的草稿）

**生成完必检**：
1. 草稿能否正常打开
2. 素材是否都能正常显示（无红色叉号）
3. 时间轴每镜时长和顺序对不对
4. 字幕不遮关键内容
5. 配音和画面是否同步

详见 `references/step-4-jianying-draft.md`。

## Final Check

每条视频跑完后必查：

- [ ] `recipe.json` 里的文案已人工 review 过（ASR/OCR 有误差）
- [ ] `matches.json` 里没有"降级匹配"标记（有的话手动补素材）
- [ ] 配音语速自然，无爆音、无跳字
- [ ] 剪映草稿能打开，素材无红叉
- [ ] 时间轴对齐（配音 = 画面）
- [ ] 字幕没有遮挡产品/关键画面

## Downstream Usage

Skill 出的是**剪映草稿**，用户后续操作：

1. 打开剪映 → 找到 `draft/<草稿名>` → 双击打开
2. 微调（改字幕字体、加转场、换配音音色）
3. 导出成 MP4 → 发抖音

Skill 不管**发布环节**（那是抖音后台的事）。

## Common Failures

见 `references/troubleshooting.md`：

- 剪映打不开草稿 → ID 冲突或路径失效
- 素材匹配都是低置信度 → 素材库分类不够细
- 配音跟画面不同步 → Step 3 节奏对齐没做
- 拆解出来的镜头都是零点几秒 → FFmpeg 场景检测阈值太低
