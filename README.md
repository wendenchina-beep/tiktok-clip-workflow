# 🎬 TikTok Clip Workflow

> 抖音/短视频自动化剪辑工作流 —— 从"参考视频 + 素材库"到"可打开的剪映草稿"全自动生成

---

## 这是什么

**TikTok Clip Workflow** 是一个为 AI 编程助手（Codex / Claude Code / MiniMax Code 等）设计的 Skill。

它解决的核心痛点是：

> 想抄爆款做视频，但一条要花两三小时手动对着参考视频找素材、卡时间、加字幕，一天撑死出 6 条。

用这套工作流，一条视频从参考到剪映草稿就 20-30 分钟，一天轻松 30 条，效率提升 5 倍以上。

## 工作原理

**核心思路：抄爆款结构 + 复用素材库**

参考视频提供**结构和节奏**，用户素材库提供**画面**，配音和字幕自动生成，最后一键出剪映草稿。

完整 4 步流水线：

```
Step 1: 拆解参考视频     → 拆成一个个镜头，提关键帧，出 recipe.json
   ↓
Step 2: 视觉匹配素材     → 从你的素材库匹配每个镜头的替换素材
   ↓
Step 3: 生成配音         → TTS 生成配音，按配音反向调整视频节奏
   ↓
Step 4: 生成剪映草稿     → 打包成剪映可以直接打开的草稿
```

**这个 Skill 只做"自动生产"，不做"精品创作"**。适合的是需要多段素材拼接的偏混剪类视频（电商带货、营销推广、混剪 Vlog）。想做高质量原创视频（复杂转场、细腻情绪）不适合。

## 能做什么

### 📐 拆解参考视频
自动识别参考视频的镜头切换点，把整段视频拆成一个个独立镜头。用 FFmpeg 场景检测 + 关键帧提取 + ASR/OCR 文案识别，输出 `recipe.json` 记录完整结构。

### 🎯 视觉匹配素材
遍历你的素材库，给每个镜头找到**最匹配**的替换素材。基于颜色、亮度、构图相似度计算匹配得分。多样性铁律防止"AI 感"过重（同一素材连续 3 镜必换、同类型场景连续 5 镜必换）。

### 🎙️ 生成配音
支持豆包语音大模型（付费但稳定，推荐带货）或 VoxCPM（免费开源，声音克隆）。配音生成后**反向调整**视频节奏——配音 5 秒，画面就拉长到 5 秒，配音短就压缩画面。

### 🎬 生成剪映草稿
直接输出剪映可以打开的草稿目录。锁死草稿三个关键 ID 一致性、素材绝对路径、时间轴毫秒级对齐——踩过的所有坑都封装了。

### 🚫 宁缺勿滥
匹配置信度 < 0.60 直接标记「缺素材」不硬填。宁可让用户手动补，也不用无关素材凑出一条看起来很怪的视频。

## 安装

将 `tiktok-clip-workflow` 文件夹放到你的 Skill 目录：

```bash
# Codex
~/.codex/skills/tiktok-clip-workflow

# MiniMax Code (Mavis)
~/.mavis/skills/tiktok-clip-workflow
```

目录结构：

```
tiktok-clip-workflow/
├── SKILL.md                          # 技能主文件（4 步流程 + 触发判定）
└── references/
    ├── environment-setup.md          # 环境搭建（FFmpeg/Python/剪映/豆包）
    ├── agents-template.md            # 项目 AGENTS.md 配置模板
    ├── step-1-decompose.md           # 拆解参考视频（recipe.json 结构）
    ├── step-2-match-assets.md        # 素材匹配（多样性铁律 + 置信度阈值）
    ├── step-3-generate-voice.md      # 配音生成（豆包/VoxCPM + 节奏对齐）
    ├── step-4-jianying-draft.md      # 剪映草稿（3 ID 一致性 + 路径规则）
    └── troubleshooting.md            # 常见问题排查（5 大类问题）
```

## 使用方法

### 4 步线性流程

```
Step 0: 准备环境和项目     → 装工具 + 建目录结构 + 写 AGENTS.md
   ↓
Step 1: 拆解参考视频       → 一句话唤起 Skill
   ↓
Step 2: 匹配素材           → Skill 从你素材库找匹配
   ↓
Step 3: 生成配音           → TTS 出配音
   ↓
Step 4: 生成剪映草稿       → 打开剪映直接微调
```

**关键理解**：这个 Skill 不生成视频文件，也不发布视频——它只生成**剪映草稿**。你后续在剪映里微调，再导出成 MP4 发抖音。

---

### Step 0：准备环境和项目

**首次使用**：跑环境自检
```
用 tiktok-clip-workflow 帮我做一次环境自检
```
Skill 会检查 FFmpeg / Python / 依赖库 / 剪映草稿目录是否就绪。缺什么按 `references/environment-setup.md` 装。

**新项目建目录**：
```
建一个新项目 <项目名>，准备好素材库目录和 AGENTS.md
```
Skill 会按标准结构 `mkdir` + 复制 AGENTS.md 模板。

标准目录：
```
项目根/
├── AGENTS.md                       # 项目配置
├── assets/                         # 素材库（永久，按类别分子目录）
└── work/                           # 工作区（每条视频一个日期文件夹）
    └── 2026-07-09/
        ├── reference.mp4
        └── ...（各步产物）
```

---

### Step 1：拆解参考视频

```
把 work/2026-07-09/reference.mp4 拆解一下
```

Skill 会：
1. 用 FFmpeg 场景检测拆分镜头 → `shots/shot_XXX.mp4`
2. 每镜提取 3 张关键帧 → `keyframes/`
3. 用 ASR/OCR 提取文案 → 挂到每镜的 text 字段
4. 输出 `recipe.json`

**⏸ 让用户 review**：镜头切分对不对、文案对不对，有问题告诉 Skill 局部调整。

---

### Step 2：匹配素材

```
从 assets/ 匹配每个镜头的素材
```

Skill 会：
1. 遍历 `assets/`，给每个素材提关键帧和视觉特征
2. 给每个镜头找匹配得分 Top 3
3. 应用多样性铁律（避免连续重复）
4. 高置信度直接采用，中置信度标记降级，低置信度标记「缺素材」
5. 复制（不是移动）到 `selected/`

**⏸ 让用户 review**：低置信度镜头人工补素材，或者让 Skill 换个素材。

---

### Step 3：生成配音

```
用豆包 BV700 音色生成配音
```

Skill 会：
1. 读 `recipe.json` 里的文案（用户可先修正）
2. 调 TTS API 生成每镜配音 → `voice/voice_XXX.mp3`
3. **反向调整视频节奏**：更新 `matches.json` 的每镜时长以匹配配音

**⏸ 让用户 review**：听一遍配音，语速/音色/读音有问题告诉 Skill。

---

### Step 4：生成剪映草稿

```
生成剪映草稿，草稿名 20260709_产品视频_01
```

Skill 会：
1. 生成新 UUID 并检查冲突（避免覆盖已有草稿）
2. 构造 `draft_content.json` + `draft_meta_info.json`
3. 素材路径全部绝对路径
4. 三个关键 ID 保持一致
5. 视频/音频/字幕轨毫秒级对齐（用微秒单位）
6. 落盘到剪映草稿目录

**⏸ 打开剪映验证**：
- 草稿能否打开
- 素材有没有红叉
- 时间轴对不对
- 字幕不遮画面
- 配音跟画面同步

## 核心原则

| 原则 | 说明 |
| --- | --- |
| 🎯 **抄爆款结构** | 参考视频提供节奏，你的素材库提供画面 |
| 🚫 **宁缺勿滥** | 置信度 < 0.60 标记缺素材，不用无关素材硬填 |
| 🔄 **多样性铁律** | 同素材/同类型/同目录 连续使用有上限，防机器感 |
| 📁 **只复制不移动** | 保护原始素材库，一律 `cp` 不 `mv` |
| 🆔 **草稿 ID 必唯一** | 新 UUID 冲突检查，避免覆盖已有草稿 |
| ⏸ **每步必 review** | 4 步流水线每步都留 ⏸ Gate 让用户看中间产物 |

## 适用场景

| 类型 | 适不适合 | 备注 |
| --- | --- | --- |
| 电商带货短视频 | ✅ 完美适合 | 作者原始场景，一天 30 条 |
| 混剪 Vlog | ✅ 适合 | 素材库准备好就行 |
| 营销推广短视频 | ✅ 适合 | 参考爆款结构最有效 |
| 短剧 / 剧情类 | ⚠️ 部分适合 | 需要人工精修镜头语言 |
| 精品广告 / TVC | ❌ 不适合 | 需要复杂转场、细腻情绪 |
| 全新原创无参考 | ❌ 不适合 | 本 Skill 靠"抄结构"驱动 |

## 输出物

Skill 的最终产出：

1. **剪映草稿目录** — 打开剪映就能看到、双击就能编辑
2. **中间产物** — `recipe.json` / `matches.json` / `voice/timing.json`（用户可以复用、迭代）
3. **配音文件** — MP3 格式，可以单独用

Skill 不管**发布环节**（那是抖音后台的事）。

## Skill 的边界（我不做什么）

**不做**：
- ❌ 不做**原创剧本**（本 Skill 靠参考视频驱动结构）
- ❌ 不做**精品级镜头语言**（需要人工，Skill 只做匹配）
- ❌ 不做**发布到抖音**（用户自己在剪映导出后发布）
- ❌ 不做**复杂转场 / 特效**（剪映草稿只含基础剪辑）
- ❌ 不做**版权判定**（用户负责素材合规）

**做**：
- ✅ 拿到"参考视频 + 素材库" → 输出可打开的剪映草稿
- ✅ 4 步流水线全自动，每步留 ⏸ Gate 让用户 review
- ✅ 踩过的所有坑（ID 冲突、路径、时间单位）都封装
